Application Menus

Guidelines

  • Limit the number of top level menus
  • Avoid blatant branding/marketing
  • Avoid using more than 2 layers of sub menus
  • Use disable/enable vs hide/show for contextual functionality

Context Menus

Guidelines

  • Contextually relevant
  • Limit functionality
  • Recognize limited discoverability
  • Consider Application menu and keyboard shortcuts
  • Should be duplicated in Application menu
  • Avoid sub menus
  • Use menu groups

Toolbar

Guidelines

  • Recommended for stateful indicators (interactive)
  • Minimal design icons (limited color) See Noun Project Guidelines
  • Avoid blatant branding/marketing
  • 100% User configurable
  • Extensions cannot create toolbars

Status Bar

not recommended

unless you have a good reason to

Panels

discouraged

Guidelines

  • Should be contextually relevant
  • Easy to hide
  • Temporary ui
  • Use dialogs for settings
  • For functionality that would use the whole panel
  • Avoid stacked panels

Dialogs

Guidelines

  • Recommended for settings
  • Potentially useful for property changes (consider inline editor)
  • Consideration: freezes ui

Inline Editor

Guidelines

  • Pertains to or effects code contextually
  • Intended primarily for editing
  • Can be used for previewing
  • Usually temporary action
  • Use a maximum height
